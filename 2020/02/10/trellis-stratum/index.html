Trellis+Stratum样例试跑及运行机制解析

试跑了一下 Trellis+Stratum样例，记录一下，为后面使用真实P4设备运行作准备。

Trellis+Stratum样例运行及解析

试跑参考Stratum项目中example步骤，具体参见stratum-trellis-example，ONOS控制器及Mininet模拟器都运行在docker容器中，容器的启停及配置(onos/mininet)都通过Makefile管理，直接使用make命令来操作。

代码下载

样例运行依赖

样例主要依赖有docker和make工具，docker使用docker-compose来启动多个容器，我这边安装版本如下:

  1. docker 19.03.5

  2. docker-compose 1.25.3

  3. make 4.1

Make命令

Make command Description
make pull 拉取样例所需docker image
make start 启动所有docker容器 (mininet and onos)
make stop 关闭所有docker容器
make onos-cli 访问ONOS命令行 (password: rocks, Ctrl+D to exit)
make onos-ui 打开ONOS ui (user onos password rocks)
make mn-cli 访问Mininet命令行 (Ctrl+A Ctrl+D to exit)
make onos-log 显示ONOS运行日志
make mn-log 显示Mininet日志 (i.e., the CLI output)
make netcfg 推送netcfg.json文件对ONOS进行配置
make reset 重置本教程环境

样例运行

  1. 启动onos控制器容器及mininet模拟器容器

onos控制器容器和mininet容器由docker-compose定义启动,启动如果报错需要注意docker及docker-compose的版本是否兼容，如下docker-compose.yml文件:

如上docker-compose定义了两个Service分别对应onos容器和mininet容器，mininet容器开放了50001~50004端口，onos容器开放了8181和8101端口分别用于ui 页面访问和cli ssh访问,onos默认开启的App有:gui,drivers.bmv2,pipelines.fabric,segmentrouting,lldpprovider,hostprovider

Trellis项目核心app segmentrouting，提供了fabric网络业务打通需要的流表下发，P4 pipeline app为pipelines.fabric，提供了P4转发面程序，并为segmentrouting下流表提供了解释器以适配Trellis项目.

  1. 网络配置

make netcfg功能为导入网络配置文件到onos控制器，核心为导入模拟转发设备(stratum-bmv2)和配置端口interface ip&vlan，配置文件为./topo/netcfg.json，导入成功，访问http://localhost:8181/onos/ui/index.html#/topo可发现模拟设备已上线，拓扑如下:

使用cli查看devices,4个设备分别通过50001~50004端口建立stratum连接,onos启动stratum客户端，设备测启动stratum的服务端.

  1. host测试

使用make mn-cli访问mininet cli，mininet模拟器创建拓扑脚本为./topo/topo.py，创建了4个stratum-bmv2转发设备，4个host终端，可进行host网络功能验证:

ping 测试

本样例未使用dhcp分配ip，host都静态配置的ip，h1和h2为同一子网，ping直接会触发arp信息交换，直接可以ping通，h1和h3为不同子网，需要保证两个host都上线，保证onos控制器已知了h3接入位置，h1和h3才能ping通.
